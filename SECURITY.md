# Security Policy

## Supported Versions

BitChaser is currently under active development. Security updates are provided only for the latest version of the `main` branch.

| Version                     | Supported |
| --------------------------- | --------- |
| Latest `main` branch        | Yes       |
| Earlier commits or releases | No        |

## Reporting a Vulnerability

Please do not report suspected security vulnerabilities through a public GitHub issue, pull request, or discussion.

Instead, report the vulnerability privately using GitHub's private vulnerability reporting or security advisory features for this repository, when available.

Include as much of the following information as possible:

* A description of the vulnerability
* The affected file, function, component, or dependency
* Steps required to reproduce the issue
* A minimal proof of concept, when appropriate
* The potential impact
* Any suggested mitigation or correction
* Your operating system and Python version

You should receive an acknowledgment within seven days. Additional updates will be provided when practical as the report is investigated.

Please allow a reasonable amount of time for the issue to be investigated and corrected before publicly disclosing it.

## Scope

Examples of issues that may qualify as security vulnerabilities include:

* Unintended arbitrary code execution
* Unsafe handling of untrusted files or data
* Path traversal or unintended file access
* Exposure of secrets or private information
* Vulnerable dependencies that materially affect BitChaser
* Security weaknesses in future network-facing functionality

Ordinary bugs, incorrect forecasts, model-performance concerns, documentation errors, and feature requests should be submitted through the public issue tracker.

## Security Limitations

BitChaser is a research project and is not intended to provide financial, investment, trading, or security guarantees.

Forecasts and model outputs should not be treated as financial advice.
